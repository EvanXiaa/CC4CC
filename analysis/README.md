## How to Contribute

### File Naming Convention

Name your analysis file as: `PR{number}_{short-title}.md`

Example: `PR42_tool-permission-flow.md`

### File Template

```markdown
# [Your Analysis Title]

> **Educational Analysis Only** - No source code included.

**Author**: [Your name/handle]
**PR**: [#XX](link-to-pr)
**Date**: YYYY-MM-DD
**Module**: [Module name]

---

## Overview

[Brief description of what this analysis covers]

---

## Analysis

[Your detailed analysis content]

---

## Key Insights

[Summary of key findings]

---

## References

- [Any references or related materials]
```

### Submission Process

1. Create your analysis file following the naming convention
2. Place it in the appropriate module's `analysis/` subdirectory
3. Update the **Extend Analysis** table in the module's main README
4. Submit a PR with your changes
